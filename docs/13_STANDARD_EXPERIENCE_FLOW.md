## Purpose
This document defines the standard participant flow for Veytec Experience Platform events and captures important data-handling requirements.

## Standard Participant Flow
1. Participant takes a picture at the kiosk or uploads an image.
2. Platform creates an Experience Record.
3. AI workflow generates one or more outputs.
4. Participant scans a QR code or opens a link to claim the image.
5. Participant fills out a claim form.
6. Platform stores the claim as a separate Claim Record.
7. Participant receives a link to their image by SMS or email.
8. Optional follow-up message is sent after the event.

## Important Data Model Rule
Claims must not overwrite the original Experience Record.

One generated image may be claimed by more than one person. Each claim should be stored as its own record linked back to the original Experience Record.

## Example
If three people scan the same QR code and submit the form, the platform should store:

- 1 Experience Record
- 3 Claim Records

Each Claim Record should include the submitted contact information and link back to the same Experience Record.

## Recommended Data Objects

### Experience Record
Represents the image/session generated by the kiosk.

Fields may include:
- Experience ID
- Event ID
- Capture timestamp
- Source image
- Generated images
- Wall visibility
- Status

### Claim Record
Represents a person claiming or requesting an image.

Fields may include:
- Claim ID
- Experience ID
- Name
- Email
- Mobile number
- SMS consent
- Email consent
- Submission timestamp
- Delivery status
- Follow-up status

## Error Handling Requirements
- If AI generation fails, show a friendly error and allow retry.
- If the claim form is submitted multiple times, create separate Claim Records.
- If delivery fails, log the failure and allow manual resend.
- If the image is not ready yet, show a waiting state instead of a broken page.
- If the original Experience Record cannot be found, show a support-friendly error message.

## Product Decision
The platform must preserve all participant submissions. No form submission should overwrite another person's data.
